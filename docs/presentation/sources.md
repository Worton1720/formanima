# Источники и научная база — шпаргалка к защите FORMANIMA

Кто, чем известен, как аргументирует проект + готовые ссылки для списка литературы диплома.

## Логика аргументации (одной картинкой)

| Звено | Источник | Что доказывает |
|---|---|---|
| **Проблема** | Лалли (2010) | Привычка формируется долго (~66 дней) → люди срываются → нужна поддержка |
| **Механизм привычки** | Дахигг (2012) | Привычка = сигнал–действие–вознаграждение; усиливаем вознаграждение |
| **Почему мотивация работает** | Деси и Райан | Игровой прогресс = чувство компетентности и автономии |
| **Как именно геймифицировать** | Ю-Кай Чоу | Очки/уровни/стрики выбраны по модели «Октализ» |
| **Надёжность решения** | OWASP, RFC 7519 | Реализация соответствует отраслевым стандартам |

Главная мысль: каждое решение в проекте обосновано — от постановки проблемы до конкретных игровых механик и безопасности.

## 1. Филиппа Лалли (Phillippa Lally) — слайд 3

- **Кто:** психолог-исследователь, Университетский колледж Лондона (UCL), психология здоровья.
- **Исследование:** «How are habits formed: Modelling habit formation in the real world» (2010). 96 участников, 12 недель.
- **Результат:** привычка становится автоматической в среднем за **66 дней** (разброс **18–254** дня).
- **Аргументирует:** опровергает миф «21 день», показывает, что удержание — длинная дистанция → нужен инструмент мотивации.
- **Деталь для вопросов:** миф про «21 день» приписывают пластическому хирургу Максвеллу Мольцу (1960-е) — он наблюдал адаптацию пациентов, а не формирование привычек.

## 2. Чарльз Дахигг (Charles Duhigg) — слайд 10

- **Кто:** журналист The New York Times, лауреат Пулитцеровской премии (2013).
- **Работа:** книга «Сила привычки» (The Power of Habit, 2012).
- **Идея — «петля привычки»:** **сигнал → действие → вознаграждение**.
- **Аргументирует:** логика приложения построена по этой петле. Сигнал — привычка в списке на сегодня; действие — выполнение; вознаграждение — очки/уровень/награда. Геймификация усиливает самое слабое звено — вознаграждение.

## 3. Эдвард Деси и Ричард Райан (Edward Deci, Richard Ryan) — слайд 10

- **Кто:** психологи, Рочестерский университет; одни из самых цитируемых в мире.
- **Теория:** самодетерминации (Self-Determination Theory, SDT), с 1980-х.
- **Идея:** мотивация бывает **внутренняя** (ради интереса) и **внешняя** (ради награды). Внутреннюю питают 3 потребности: **автономия, компетентность, связанность**.
- **Аргументирует:** объясняет, *почему* геймификация работает. Уровни/опыт → компетентность; свободный выбор целей → автономия; будущие соцрейтинги → связанность.
- **Деталь для вопросов:** SDT предупреждает об «эффекте сверхоправдания» — избыток внешних наград может вытеснять внутренний интерес. Поэтому геймификация — поддержка на старте, а не единственный двигатель.

## 4. Ю-Кай Чоу (Yu-kai Chou) — слайд 10

- **Кто:** предприниматель, пионер-теоретик геймификации (Кремниевая долина).
- **Работа:** фреймворк «Октализ» (Octalysis); книга «Actionable Gamification» (2015).
- **Идея:** вовлечённость держится на **8 движущих силах**, в т.ч. №2 «Развитие и достижения», №8 «Избегание потери».
- **Аргументирует:** механики выбраны по модели, а не наугад. Очки/уровни/награды = драйвер №2 (достижения); стрики (серии без пропусков) = драйвер №8 (страх потерять прогресс).

## 5. OWASP (Open Worldwide Application Security Project) — слайды 5 и 7

- **Кто:** международное некоммерческое сообщество; задаёт отраслевые стандарты безопасности веб-приложений.
- **Документы:** OWASP Top 10 (топ уязвимостей), Password Storage Cheat Sheet (хранение паролей).
- **Аргументирует:** пароли не хранятся в открытом виде, а хешируются (bcrypt) — прямая рекомендация OWASP. При краже базы исходные пароли восстановить нельзя.

## 6. Технические стандарты и НПБ — слайды 5, 7, 9 (для технических вопросов)

- **JWT / RFC 7519** — спецификация токенов («электронных пропусков»): приложение узнаёт пользователя без повторного ввода пароля.
- **REST** — общепринятый стиль обмена данными между интерфейсом и сервером.

**Нормативно-правовая база (РФ) — для слайда 5:**

- **ФЗ-152 «О персональных данных»** — приложение хранит e-mail и пароль пользователя → обязано защищать персональные данные (реализовано хешированием паролей, разграничением доступа).
- **ФЗ-149 «Об информации, информационных технологиях и о защите информации»** — общие требования к обработке и защите информации.
- **ГОСТ 34.601-90** — стадии создания автоматизированных систем (методическая основа этапов работы).
- **ГОСТ 34.602-89** — техническое задание на создание АС (основа постановки задачи и требований).
